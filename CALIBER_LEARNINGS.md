- **[gotcha]** When reading GitHub Actions logs for an in-progress run, `gh run view --job <id> --log-failed` (and `--log`) returns `run ... is still in progress; logs will be available when it is complete`. To get logs of a still-running job, use `gh api repos/{owner}/{repo}/actions/jobs/{jobId}/logs` instead — that streams partial logs immediately.

- **[gotcha]** The "Codacy Static Code Analysis" PR check can report `fail` but does NOT block merge — `master` is UNPROTECTED (`gh api repos/detain/phlix-windows-client/branches/master/protection` returns 404 "Branch not protected"), so Codacy is advisory-only. - **[gotcha]** After `npm audit fix` reports `found 0 vulnerabilities` locally, `gh api repos/.../dependabot/alerts` still shows the alerts OPEN — Dependabot only closes them after the fix is merged to the default branch and re-scanned. Don't read the unchanged alert count as a failed fix; verify locally with `npm audit` + `npm ls <pkg>` instead.
- **[fix]** For Dependabot vite/vitest alerts here: old `vitest@1.x` + `@vitest/coverage-v8@1.x` transitively pull a vulnerable `vite@5.4`. Bump both with `npm pkg set devDependencies.vitest="^3.2.6" devDependencies.@vitest/coverage-v8="^3.2.6"`, then `npm install` + `npm audit fix` (NOT `--force`) clears the chain to 0 vulns; the 22-test suite, coverage, build, lint, and vue-tsc all stay green under vitest 3.
- **[gotcha]** vitest coverage `exclude` in `vitest.config.mts` must cover ALL root config files, not just `vite.config.mts` — otherwise `vitest.config.mts` / `eslint.config.mjs` count as 0%-covered source and tank the "All files" total (showed 48% instead of 100%). Use the glob `**/*.config.{ts,mts,js,mjs,cjs}` rather than naming individual files.
- **[gotcha]** Windows CI `EBADENGINE` warnings (`electron@42` / `@electron/get` / `@electron/rebuild` / `@babel 8` all want node `>=22.12`) are WARNINGS, not failures — `npm install` still succeeds and the job passes. They appear in the `Install dependencies` step log on Node 20 runners. The fix is to bump the runner Node, not to chase the install exit code.
- **[convention]** CI Node version for phlix repos is pinned in TWO files: `.github/workflows/test.yml` (`node-version: [NN.x]` matrix + lint/type-check jobs + the `runner.os == 'ubuntu-latest' && matrix.node-version == 'NN.x'` coverage-upload conditional) and `.github/workflows/build.yml` (`NODE_VERSION: 'NN.x'` env). Bump BOTH, and when changing the matrix version remember the coverage-upload `if:` conditional hard-codes the same string. Node 24 is the org-preferred version (satisfies electron >=22.12 and @babel 8 / vitest 3 >=24.11).
- **[gotcha]** When bumping a `github:detain/phlix-ui#vX.Y.Z` (or any git-tag) dependency, `npm pkg set` + plain `npm install` can silently KEEP the old version installed (lockfile/cache resolves the old tag) — verify with `node -p "require('./node_modules/@phlix/ui/package.json').version"`. If it's stale, force re-resolve with `npm install "@phlix/ui@github:detain/phlix-ui#vX.Y.Z" --save`, then re-check the installed version AND that the new seam exports exist (`grep -oE 'seekBy|playLocalFile' node_modules/@phlix/ui/dist/index.d.ts`).
