# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.

- **[gotcha]** Windows CI `EBADENGINE` warnings (`electron@42` / `@electron/get` / `@electron/rebuild` / `@babel 8` all want node `>=22.12`) are WARNINGS, not failures — `npm install` still succeeds and the job passes. They appear in the `Install dependencies` step log on Node 20 runners. The fix is to bump the runner Node, not to chase the install exit code.
- **[convention]** CI Node version for phlix repos is pinned in TWO files: `.github/workflows/test.yml` (`node-version: [NN.x]` matrix + lint/type-check jobs + the `runner.os == 'ubuntu-latest' && matrix.node-version == 'NN.x'` coverage-upload conditional) and `.github/workflows/build.yml` (`NODE_VERSION: 'NN.x'` env). Bump BOTH, and when changing the matrix version remember the coverage-upload `if:` conditional hard-codes the same string. Node 24 is the org-preferred version (satisfies electron >=22.12 and @babel 8 / vitest 3 >=24.11).
- **[gotcha]** When bumping a `github:detain/phlix-ui#vX.Y.Z` (or any git-tag) dependency, `npm pkg set` + plain `npm install` can silently KEEP the old version installed (lockfile/cache resolves the old tag) — verify with `node -p "require('./node_modules/@phlix/ui/package.json').version"`. If it's stale, force re-resolve with `npm install "@phlix/ui@github:detain/phlix-ui#vX.Y.Z" --save`, then re-check the installed version AND that the new seam exports exist (`grep -oE 'seekBy|playLocalFile' node_modules/@phlix/ui/dist/index.d.ts`).
- **[convention]** The renderer build (`tsconfig.json`) is not clean on master; a green `npm test` (vitest) is the real pre-commit signal for renderer changes, not a zero-error `tsc`. Post Vue/`@phlix/ui` migration the suite is `tests/unit/{resolveConfig,electronBridge,main}.test.ts` (now 23 tests after the seekBy bridge wiring) and `npx eslint .` exits 0 — run both (`npx eslint . && npx vitest run`) as the green gate before staging. Full local PR-gate command: `npx vue-tsc --noEmit && npx tsc -p tsconfig.main.json --noEmit && npx vitest run && npx eslint .` (all four return EXIT=0 on a clean change).
- **[gotcha]** A backgrounded CI-poll loop (`for i in $(seq …); do s=$(gh pr checks N); … break; done`) exits with the last command's status, so when Codacy (or any check) shows `fail` the loop's final `gh pr checks` exits non-zero and the `<task-notification>` reports the background command as `failed` — even though CI actually settled fine. Always `Read` the task's output file before reacting; the "failed" status is just the trailing exit code, not a real CI failure.
- **[fix]** For Dependabot vite/vitest alerts here: old `vitest@1.x` + `@vitest/coverage-v8@1.x` transitively pull a vulnerable `vite@5.4`. Bump both with `npm pkg set devDependencies.vitest="^3.2.6" devDependencies.@vitest/coverage-v8="^3.2.6"`, then `npm install` + `npm audit fix` (NOT `--force`) clears the chain to 0 vulns; the suite, coverage, build, lint, and vue-tsc all stay green under vitest 3.
- **[pattern]** To clear deprecated transitive deps (the `inflight` memory-leak warning, old `glob`/`rimraf`) without touching electron-builder, add an `overrides` block to `package.json` (`{"glob":"^13.0.6","rimraf":"^6.1.3"}`) and `npm install` — modern glob/rimraf dropped `inflight`. Verify with `npm ls inflight` returning `└── (empty)` and `npm ls glob --all | grep -oE 'glob@[0-9]+'` showing only the new major, then re-run `npx vitest run` to confirm electron-builder's asar/temp paths still resolve. `npm warn deprecated glob@11.x` can still print during install even after the override resolves correctly — check the actual installed tree, not the warning.

- **[gotcha]** GitHub Actions artifacts are ALWAYS served zipped — there is no setting to offer a bare clickable file. To provide a raw downloadable `.exe`/`.appx`, you must attach it to a GitHub **Release** (e.g. a rolling "latest" prerelease), not an upload-artifact step.
- **[pattern]** Rolling "latest" release in `build.yml`: a `release-latest` job (gated `if: github.event_name == 'push' && github.ref == 'refs/heads/master'`) that `actions/download-artifact`s the built installers, runs `gh release delete latest --yes --cleanup-tag || true` (so the tag re-points to the new commit and no stale assets accumulate), then `softprops/action-gh-release@v3` with `tag_name: latest`, `prerelease: true`, `make_latest: false`, `target_commitish: ${{ github.sha }}`. The job needs `permissions: contents: write`.
- **[convention]** `build.yml` has three-way gating: the `build` job runs on push/PR/tag; the `release-latest` job runs ONLY on master pushes (skips PRs and tags); the `release` (versioned) job runs ONLY on `refs/tags/v*`. Preserve these guards when adding triggers so PR/master builds publish downloadable assets without cutting a versioned release.
